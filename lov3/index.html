1. 프로젝트 개요

LOV3는 데이트 장소를 기반으로 동행자를 모집하고 매칭하는 게시판 서비스입니다. 사용자는 특정 장소를 선택해 게시글을 작성하고, 관심 있는 상대방에게 데이트 신청을 보낼 수 있습니다.

Spring Boot 3 기반의 REST API 서버를 직접 설계·구현하고, JWT 인증, JPA 연관관계, Docker 기반 배포까지 전 과정을 경험하는 것을 목표로 합니다.

2. 요구사항 분석

기능 요구사항

비회원은 게시글·장소 조회만 가능
로그인 회원은 게시글 작성, 데이트 신청 가능
게시글 작성자만 본인 글 수정·삭제 가능 (관리자 포함)
데이트 신청은 게시글 작성자만 수락·거절 가능
관리자만 장소 등록·수정·삭제 가능
게시글은 반드시 기존 장소 중 하나를 선택해야 함

비기능 요구사항

인증은 Stateless JWT 방식으로 처리 (세션 미사용)
모든 비밀번호 BCrypt 암호화 저장
이미지 파일은 서버 파일시스템에 저장, Docker 볼륨으로 영속성 보장
ddl-auto=none — DB 스키마 변경은 수동 관리
CORS 전체 허용 (개발/운영 통합)

6. API 설계

회원 (Member)

MethodURL인증설명
POST/login—로그인 (JWT 발급)
POST/members—회원가입
GET/members/check?id=—ID 중복 확인
GET/members/my✅내 정보 조회
PUT/members/my✅내 정보 수정
DEL/members/my✅회원 탈퇴

게시판 (Board)

MethodURL인증설명
GET/boards?page=0&size=10—전체 게시글 (페이징)
GET/boards/{boardId}—게시글 상세
GET/boards/my✅내 게시글
GET/boards/search?keyword=—키워드 검색
POST/boards✅게시글 등록
PUT/boards/{boardId}✅게시글 수정 (작성자/관리자)
DEL/boards/{boardId}✅게시글 삭제

데이트 신청 (DateRequest)

MethodURL인증설명
POST/boards/{boardId}/date-requests✅데이트 신청
GET/date-requests/received✅받은 신청 전체
GET/date-requests/sent✅보낸 신청 전체
PUT/date-requests/{id}/accept✅신청 수락 (경쟁 신청 자동 거절)
PUT/date-requests/{id}/reject✅신청 거절
DEL/date-requests/{id}✅신청 취소

장소 (Place)

MethodURL인증설명
GET/places—전체 장소
GET/places/search?keyword=—장소 검색
POST/places🔒 ADMIN장소 등록
PUT/places/{placeId}🔒 ADMIN장소 수정
DEL/places/{placeId}🔒 ADMIN장소 삭제
응답 형식: JSON  ·  인증 헤더: Authorization: Bearer {token}  ·  성공 코드: 200 / 201 / 204  ·  로그인: application/x-www-form-urlencoded

15. 트러블슈팅
N+1 문제 — 게시글 목록 조회 시 쿼리 폭발
원인LAZY 로딩 상태에서 게시글 목록 조회 시 Member, Place를 각각 추가 쿼리로 로딩
해결JOIN FETCH 쿼리로 한 번에 로딩 + 페이징 COUNT 쿼리를 countQuery로 분리해 불필요한 JOIN 방지
교훈연관관계 조회가 많은 API는 초기 설계 단계에서 Fetch 전략을 함께 고려해야 함
Mac(NFD) ↔ Linux(NFC) 유니코드 정규화 불일치
원인한글 파일명이 macOS(NFD)에서 정상 업로드 → Linux EC2(NFC) 환경에서 파일 참조 실패
해결업로드 시 파일명을 UUID로 변환해 인코딩 차이를 원천 차단
교훈개발 환경(Mac)과 운영 환경(Linux)의 파일시스템 차이를 항상 고려해야 함
컨테이너 재시작 시 업로드 이미지 소실
원인업로드 이미지가 컨테이너 내부 파일시스템에만 저장되어 재시작 시 데이터 소실
해결docker-compose에 Named Volume(upload-images) 마운트로 호스트에 영속성 확보
교훈컨테이너는 Stateless — 영속 데이터는 반드시 Volume으로 분리해야 함
