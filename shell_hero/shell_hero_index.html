기술적 도전
01 · 전투 데이터 격리 (Snapshot 패턴)

전투 중 임시 버프·HP/MP 변동이 세션 원본을 오염시키는 문제를 방지하기 위해, 전투 시작 시 BattleHeroDTO로 원본을 스냅샷해 사용하고 결과(승/패)에 따라 선택적으로 세션에 반영했습니다.

// 원본 HeroDTO → 전투 전용 BattleHeroDTO 복사 (격리) public BattleHeroDTO createBattleHero() { HeroDTO hero = LoginSession.getInstance().getCurrentHero(); return new BattleHeroDTO( hero.getHeroName(), hero.getHeroLevel(), hero.getHeroExp(), hero.getHeroHp(), hero.getHeroMp(), hero.getHeroAttack(), hero.getHeroDefense() ); }
02 · 복합 트랜잭션 처리

전투 보상 처리 시 Hero·Gem·업적·스테이지 4개 테이블을 단일 트랜잭션으로 묶어, 중간 실패 시 전체 rollback으로 데이터 정합성을 보장했습니다.

Connection con = DBManager.getConnection(); con.setAutoCommit(false); try { heroService.updateHero(con, hero); // 영웅 정보 갱신 heroService.updateHeroGem(con, heroId, gem); // Gem 갱신 questService.updateQuestProgress(con, quest); // 업적 갱신 heroService.updateClearStage(con, heroId, stage); // 스테이지 갱신 con.commit(); } catch (Exception e) { con.rollback(); throw new GameException("보상 처리 중 오류 발생"); } finally { con.setAutoCommit(true); DBManager.close(con); }
03 · DAO 인터페이스-구현체 분리

모든 DAO를 Interface / Impl로 분리해 느슨한 결합을 확보하고, 향후 ORM 전환이나 테스트 Mock 주입이 용이한 구조로 설계했습니다.
